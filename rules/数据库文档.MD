## 表结构详情

### users 用户表

存储用户基本信息。

```sql
CREATE TABLE users (
  id              VARCHAR(36) PRIMARY KEY,
  username        VARCHAR(50) NOT NULL UNIQUE,
  nickname        VARCHAR(100) NOT NULL,
  avatar          VARCHAR(500),
  phone           VARCHAR(20) NOT NULL UNIQUE,
  password_hash   VARCHAR(255) NOT NULL,
  email           VARCHAR(100),
  location        VARCHAR(100),
  location_code   VARCHAR(12),
  gender          TINYINT DEFAULT NULL COMMENT '0=未知, 1=男, 2=女',
  birthday        DATE DEFAULT NULL,
  status          ENUM('active', 'banned', 'deleted') DEFAULT 'active',
  points          BIGINT DEFAULT 0,
  last_login_at   DATETIME,
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_phone (phone),
  INDEX idx_status (status)
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| username | VARCHAR(50) | 用户名（唯一） |
| nickname | VARCHAR(100) | 昵称 |
| avatar | VARCHAR(500) | 头像 URL |
| phone | VARCHAR(20) | 手机号（唯一） |
| password_hash | VARCHAR(255) | bcrypt 加密后的密码 |
| email | VARCHAR(100) | 邮箱 |
| location | VARCHAR(100) | 所在地区（文本，如"四川省 成都市 锦江区"） |
| location_code | VARCHAR(12) | 所在地区代码（GB/T 2260，如 510104） |
| gender | TINYINT | 性别：0=未知, 1=男, 2=女 |
| birthday | DATE | 生日 |
| status | ENUM | 状态：active/banned/deleted |
| points | BIGINT | 积分余额 |
| last_login_at | DATETIME | 最后登录时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### areas 行政区划表

GB/T 2260 中国行政区划代码表，支持省→市→区县三级联动。

```sql
CREATE TABLE areas (
  id          VARCHAR(12) PRIMARY KEY COMMENT 'GB/T 2260 代码',
  name        VARCHAR(50) NOT NULL,
  level       TINYINT NOT NULL COMMENT '1=省/直辖市/自治区, 2=市, 3=区县',
  parent_id   VARCHAR(12) DEFAULT NULL,
  INDEX idx_level (level),
  INDEX idx_parent (parent_id)
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(12) | GB/T 2260 行政区划代码 |
| name | VARCHAR(50) | 行政区划名称 |
| level | TINYINT | 级别：1=省级, 2=地级, 3=县级 |
| parent_id | VARCHAR(12) | 父级代码（省级为 NULL） |

**示例数据：**

| id | name | level | parent_id |
|----|------|-------|-----------|
| 510000 | 四川省 | 1 | NULL |
| 510100 | 成都市 | 2 | 510000 |
| 510104 | 锦江区 | 3 | 510100 |

### points_records 积分记录表

> ⚠️ **重要变更（2026-04-27）**：
> - 表名从 `point_transactions` 统一为 `points_records`
> - 新增 `action`、`balance_before`、`balance_after`、`operator_type`、`operator_id`、`related_type`、`idempotent_key` 字段
> - 字段命名统一：`type` → `business_type`，`amount` → `points`（正数）

记录用户积分的每一笔变动，包含余额快照，保证可追溯可对账。

```sql
CREATE TABLE points_records (
  id              VARCHAR(36) PRIMARY KEY,
  user_id         VARCHAR(36) NOT NULL,
  business_type   VARCHAR(50) NOT NULL COMMENT '业务类型',
  action          VARCHAR(20) NOT NULL COMMENT 'add(增加)/reduce(减少)',
  points          INT NOT NULL COMMENT '变动积分（正数）',
  balance_before  BIGINT NOT NULL COMMENT '变动前余额',
  balance_after   BIGINT NOT NULL COMMENT '变动后余额',
  operator_type   VARCHAR(20) NOT NULL DEFAULT 'system' COMMENT 'user/system/admin',
  operator_id     VARCHAR(36) DEFAULT NULL COMMENT '操作者ID',
  description     VARCHAR(255) COMMENT '变动描述',
  related_type    VARCHAR(50) DEFAULT NULL COMMENT '关联业务类型',
  related_id      VARCHAR(36) DEFAULT NULL COMMENT '关联业务ID',
  idempotent_key  VARCHAR(64) DEFAULT NULL COMMENT '幂等键，防重复扣积分',
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_business_type (business_type),
  INDEX idx_idempotent (idempotent_key),
  INDEX idx_created (created_at)
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| user_id | VARCHAR(36) | 用户 ID |
| business_type | VARCHAR(50) | 业务类型 |
| action | VARCHAR(20) | 动作：add(增加)/reduce(减少) |
| points | INT | 变动积分（正数） |
| balance_before | BIGINT | 变动前余额 |
| balance_after | BIGINT | 变动后余额 |
| operator_type | VARCHAR(20) | 操作者类型：user/system/admin |
| operator_id | VARCHAR(36) | 操作者 ID |
| description | VARCHAR(255) | 变动描述 |
| related_type | VARCHAR(50) | 关联业务类型 |
| related_id | VARCHAR(36) | 关联业务 ID |
| idempotent_key | VARCHAR(64) | 幂等键 |
| created_at | DATETIME | 创建时间 |

**业务类型（business_type）说明：**

| business_type | 说明 | action | 积分变化 |
|------|------|--------|---------|
| register | 新用户注册 | add | +100 |
| sign | 每日签到 | add | +10 |
| video | 完整观看视频 | add | +N |
| like | 点赞（视频/帖子） | add | +1 |
| comment | 发布评论 | add | +2 |
| exchange | 积分兑换商品 | reduce | -N |
| recharge | 管理员充值 | add | +N |
| adjust | 管理员调整 | reduce | -N |

### orders 订单表

```sql
CREATE TABLE orders (
  id              VARCHAR(36) PRIMARY KEY,
  user_id         VARCHAR(36) NOT NULL,
  product_id      VARCHAR(36) NOT NULL,
  order_no        VARCHAR(50) NOT NULL UNIQUE,
  quantity        INT DEFAULT 1,
  total_points   INT NOT NULL,
  status          VARCHAR(20) DEFAULT 'pending' COMMENT 'pending/paid/shipped/completed/cancelled',
  address         TEXT COMMENT '收货地址JSON',
  phone           VARCHAR(20),
  receiver_name   VARCHAR(100),
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_order_no (order_no),
  INDEX idx_status (status)
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| user_id | VARCHAR(36) | 下单用户 ID |
| product_id | VARCHAR(36) | 商品 ID |
| order_no | VARCHAR(50) | 订单号（唯一） |
| quantity | INT | 兑换数量 |
| total_points | INT | 消耗积分总额 |
| status | VARCHAR(20) | 状态：pending→paid→shipped→completed 或 cancelled |
| address | TEXT | 收货地址 |
| phone | VARCHAR(20) | 收货人电话 |
| receiver_name | VARCHAR(100) | 收货人姓名 |
| created_at | DATETIME | 下单时间 |
| updated_at | DATETIME | 更新时间 |

**订单状态流转：**

```
pending（待支付）→ paid（已支付）→ shipped（已发货）→ completed（已完成）
       ↓
  cancelled（已取消）
```

### messages 消息表

IM 即时通讯消息记录。

```sql
CREATE TABLE messages (
  id              VARCHAR(36) PRIMARY KEY,
  from_user_id    VARCHAR(36) NOT NULL,

  is_read         TINYINT DEFAULT 0,
  created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_from (from_user_id),
  INDEX idx_to (to_user_id),
  INDEX idx_created (created_at)
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| from_user_id | VARCHAR(36) | 发送者用户 ID |
| to_user_id | VARCHAR(36) | 接收者用户 ID |
| content | TEXT | 消息内容 |
| type | VARCHAR(20) | 消息类型：text/image/video/system |
| is_read | TINYINT | 是否已读：0=未读, 1=已读 |
| created_at | DATETIME | 发送时间 |
