## 接口规范

### 通用规则

- **基础 URL**：
  - 开发环境：`http://localhost:4000`
  - 生产环境：`/api`（通过 nginx 代理到 gateway）
- **认证方式**：JWT Token，在请求头中传递
  ```
  Authorization: Bearer <token>
  ```
- **响应格式**：
  ```json
  {
    "code": 0,        // 0=成功，其他=失败
    "data": {},        // 响应数据
    "message": "success" // 消息
  }
  ```
- **错误码**：
  - `0` - 成功
  - `400` - 参数错误
  - `401` - 未授权
  - `403` - 无权访问
  - `404` - 资源不存在
  - `500` - 服务器错误

### 服务端口分配

| 服务 | 端口 | 说明 |
|------|------|------|
| gateway | 4000 | API 网关，统一入口 |
| user-service | 4001 | 用户服务 |
| video-service | 4002 | 视频服务 |
| points-service | 4003 | 积分服务 |
| social-service | 4004 | 社交服务 |
| shop-service | 4005 | 商城服务 |
| im-service | 8080 | WebSocket 即时通讯 |

## Gateway 网关

**基础路径**：`/api`

网关不处理具体业务，只负责路由转发。

### 健康检查

```
GET /health
```

响应：
```json
{
  "status": "ok",
  "timestamp": "2026-04-26T10:00:00.000Z"
}
```

### 路由规则

| 前端路径 | 转发目标 | 说明 |
|----------|----------|------|
| `/api/user/*` | user-service:4001 | 用户服务 |
| `/api/video/*` | video-service:4002 | 视频服务 |
| `/api/points/*` | points-service:4003 | 积分服务 |
| `/api/social/*` | social-service:4004 | 社交服务 |
| `/api/shop/*` | shop-service:4005 | 商城服务 |

## User Service 用户服务

**服务路径**：`/api/user`
**目标服务**：`http://localhost:4001`

### 1. 发送验证码

```
POST /api/user/send-code
```

请求体：
```json
{
  "phone": "13800138000"
}
```

响应：
```json
{
  "code": 0,
  "data": { "code": "123456" },
  "message": "验证码已发送（开发模式直接返回验证码）"
```

### 6. 获取地区列表（行政区划）

```
GET /api/user/areas
```

查询参数：
- `level` - 级别：1=省级, 2=市级, 3=区县
- `parent_id` - 父级代码（查询市级时传省级代码，查询区县时传市级代码）

示例（获取所有省份）：
```
GET /api/user/areas?level=1
```

响应：
```json
{
  "code": 0,
  "data": [
    { "id": "110000", "name": "北京市", "level": 1, "parent_id": null },
    { "id": "510000", "name": "四川省", "level": 1, "parent_id": null },
    ...
  ],
  "message": "success"
}
```

示例（获取四川省下的市）：
```
GET /api/user/areas?parent_id=510000
```

示例（获取成都市下的区县）：
```
GET /api/user/areas?parent_id=510100
```

## IM Service 即时通讯服务

**服务类型**：WebSocket
**端口**：8080

### 连接

```
ws://localhost:8080?token=<jwt_token>
```

认证成功后会收到：
```json
{
  "type": "connected",
  "payload": {
    "socketId": "uuid",
    "userId": "uuid"
  }
}
```

---

### 客户端发送消息

#### 1. 发送消息

```json
{
  "type": "send_message",
  "payload": {
    "conversationId": "uuid",
    "content": "消息内容",
    "receiverId": "uuid",
    "msgType": "text"
  }
}
```

> 支持的 msgType：`text` / `image` / `video` / `voice` / `file`

#### 2. 标记已读

```json
{
  "type": "mark_read",
  "payload": {
    "conversationId": "uuid"
  }
}
```

#### 3. 正在输入

```json
{
  "type": "typing",
  "payload": {
    "conversationId": "uuid",
    "isTyping": true
  }
}
```

#### 4. 心跳

```json
{ "type": "ping" }
```
